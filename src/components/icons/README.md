# Icon set

30 custom icons generated by **Gemini 2.5 Pro** from the Orkasa logo's
geometric language: an off-axis quadrilateral outline + one signal-orange
square accent (`#FF4F1F`). Designed to feel hand-drawn but precise — the
opposite of generic icon libraries.

## Usage

```tsx
import { HomeIcon, BuildingIcon, BellIcon } from '@/components/icons/icons'

<HomeIcon size={16} className="text-ink" />
<BellIcon size={20} className="text-signal" />
```

Every icon:
- Inherits its line color from the parent text color (`currentColor`).
- Keeps the orange accent locked at `#FF4F1F` regardless of theme — the
  brand mark is intentionally non-themeable so the icon set always carries
  the Orkasa accent.
- Renders crisp at any size (24×24 viewBox, single 1.5px stroke).

## Regenerating

```bash
# All icons
npx tsx scripts/generate-icons.ts

# A single surface
npx tsx scripts/generate-icons.ts sidebar

# Specific icons
npx tsx scripts/generate-icons.ts home bell

# Then rebuild the React module
npx tsx scripts/build-icons-tsx.ts
```

The generator validates each output: viewBox must be `0 0 24 24`, exactly
one orange element, currentColor stroke, no `<text>` elements. Failed icons
print a warning but are still written so you can inspect them.

## Tuning

`scripts/generate-icons.ts` holds the style brief and the icon spec list.
To change the design language (e.g. thicker stroke, different accent
color, alternative motif), edit `STYLE_BRIEF` and re-run.

To add a new icon, append to the `ICONS` array, then run the generator
with that one name.
